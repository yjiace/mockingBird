API 文档中心

欢迎使用 MockingBird API。本文档旨在提供一个全面、清晰的指南，帮助您快速理解和使用我们的API服务。所有接口均遵循RESTful设计原则，并以JSON格式返回数据。

💡

基础URL: 您的服务部署地址, 例如: https://your-worker.your-subdomain.workers.dev

通用响应结构: 所有API请求均会返回一个包含 code, message, 和 data 字段的JSON对象。

用户管理

提供对用户资源的全面管理，包括增、删、改、查等操作。

GET /api/user

获取用户列表，支持分页和条件筛选。

查询参数

参数类型说明
pageInteger页码，默认为 1。
sizeInteger每页数量，默认为 10。
nameString按用户名称筛选。
emailString按邮箱地址筛选。
mobileString按手机号码筛选。

请求示例

curl -X GET "/api/user?page=1&size=5"

响应示例

{
  "code": 200,
  "message": "success",
  "data": {
    "results": [
      {
        "id": 1,
        "name": "示例用户",
        "email": "user@example.com",
        ...
      }
    ],
    "total": 100,
    "page": 1,
    "size": 5
  }
}
GET /api/user/{id}

通过用户ID获取指定用户的详细信息。

路径参数

参数类型说明
idInteger用户的唯一标识符。

请求示例

curl -X GET "/api/user/1"
POST /api/user

创建一个新用户。

请求体 (JSON)

{
  "name": "新用户",
  "age": 30,
  "email": "new.user@example.com",
  "mobile": "13800138000",
  "address": {
    "province": "广东",
    "city": "深圳",
    "street": "科技园路"
  }
}

响应示例

{
  "code": 200,
  "message": "success",
  "data": {
    "id": 101
  }
}
PUT /api/user/{id}

更新指定ID用户的信息。

请求体 (JSON)

{
  "name": "更新后的用户",
  "age": 31
}
DELETE /api/user/{id}

删除指定ID的用户。

POST /api/user/batch

通过一个请求批量创建多个用户。

请求体 (JSON Array)

[
  { "name": "批量用户1", "email": "batch1@example.com" },
  { "name": "批量用户2", "email": "batch2@example.com" }
]
DELETE /api/user/batch

根据提供的ID列表，批量删除用户。

请求体 (JSON)

{
  "ids": [10, 11, 12]
}
POST /api/user/init

初始化用户数据。此操作会清空现有用户并载入预设数据，主要用于测试和演示。

SSE 实时流

部分接口支持通过 Server-Sent Events (SSE) 进行流式响应。当客户端请求头中包含 Accept: text/event-stream 时，服务端将以流的形式持续推送数据。

错误处理

API在遇到错误时会返回统一格式的错误信息，方便客户端进行处理。

{
  "code": 500,
  "message": "Internal Server Error",
  "data": {
    "error": "D1_ERROR: UNIQUE constraint failed: t_user.email"
  }
}

认证机制

当前版本的API为开放访问，未设置复杂的认证机制。未来的版本可能会引入API Key或OAuth2.0等认证方式。
